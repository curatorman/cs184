Assignment 4: Inverse Kinematics

For this assignment we implemented inverse kinematics. The cube in the animation demonstrates the end point that the arm is trying to reach. In terms of file structure, our main running code is found in example_04.cpp. We use the Eigen library for matrix operations and calculating SVD. For our algorithm, we used the algorithm described in the section slides to implement inverse kinematics. We calculated the Jacobian using concatenated coordinate transformations, and then used the SVD to calculate the pseudo-inverse. Given the pseudo-inverse, we used gradient descent to find the angles of the arms that would result in the end effector being in the desired location. For the step size, we used the Armijo Backtracking line search algorithm to find an appropriate step size. Once we found a valid step size, we updated the arm angles with the difference and continued the process until we were close enough to the goal. We are also robust to out-of-reach locations by computing whether or not it is possible to reach that location given our arm parameters, and if not, then generating the arm that would be the closest to the goal.

All the different paths in the animations were created by manually toggling them on and off in the code.

For multiple arms, you must uncomment that part in the code to activate it.

Press 'c' to toggle the arm following the cursor.

When in cursor mode, press 'w' to increase the z values, and 's' to decrease them.
